- Git 的基本流程是：检查状态、暂存修改、提交记录、同步远程仓库。
- 笔记目录需要稳定，文件名需要能说明日期、论文、实验或主题。
- 学习计划应包含时间、行动和可检查的产出，而不只是宽泛目标。

## 遇到的问题

- 问题现象：命令行推送时本机 `git-remote-https.exe` 崩溃，无法完成 HTTPS 传输。
- 已尝试方法：确认仓库权限和网络连接，保留本地 Git 提交，并改用 GitHub 官方 API 发布同一批文件。
- 当前结论：问题来自本机 Git HTTPS 组件，不是仓库权限；API 同步后已检查远程 `main` 的文件清单。
- 后续处理：升级或重新安装 Git for Windows 后，再恢复常规 `git push` 工作流。

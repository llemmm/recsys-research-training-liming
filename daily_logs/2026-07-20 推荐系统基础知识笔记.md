
## 一、推荐系统全景

### 1.1 现实背景：信息过载

互联网场景下，物品、内容和服务的数量远超用户主动浏览能力，用户难以逐一比较所有候选。由此诞生两类信息筛选工具：

- **搜索引擎**：根据用户输入的查询返回相关结果，适合需求明确的场景，是「用户主动获取」
    
- **推荐系统**：根据用户、物品和场景信息主动筛选候选，帮助用户降低选择成本，是「系统自动推送」
    

### 1.2 搜索与推荐的核心区别

|   |   |   |
|---|---|---|
|维度|搜索|推荐|
|出发点|从用户明确的查询出发|从历史行为、物品信息、上下文和群体行为出发|
|需求响应|主要响应用户当前表达的明确需求|可以发现用户尚未明确表达的潜在兴趣|
|交互方式|用户主动发起|系统被动推送|

注：实际产品中二者并非完全分离，搜索结果可以个性化排序，推荐系统也会内置检索模块。

### 1.3 个性化与长尾效应

- **长尾现象**：少数热门物品获得大量曝光，大量长尾小众物品难以被用户发现。
    
- **个性化推荐的价值**：不同用户看到不同的候选列表，让长尾内容匹配到潜在兴趣用户，提升整体分发效率。
    
- **潜在问题**：热门偏置可能被进一步强化，推荐结果可能造成兴趣收窄或信息茧房。
    

### 1.4 推荐系统的分类维度

#### 1.4.1 应用领域

不同领域的数据密度、实时性和评价目标差异显著：

|   |   |   |
|---|---|---|
|应用领域|推荐对象|常见反馈或目标|
|电商|商品、店铺、搭配|点击、加购、购买|
|内容|新闻、视频、音乐、播客|播放、停留、互动|
|社交与信息流|帖子、用户、话题|关注、互动、停留|
|广告|广告和创意|点击、转化、商业约束|
|位置与服务|POI、餐饮、路线|到店、预订、时空相关性|
|知识与科研|论文、课程、知识点|相关性、发现价值、学习效果|

#### 1.4.2 推荐任务形式

- **评分预测**：预测用户会给物品打多少分
    
- **点击 / 交互预测**：预测用户是否会发生某种行为
    
- **Top-N 推荐**：从候选物品中形成排序列表
    
- **下一物品预测**：根据历史行为预测下一次交互
    
- 其他任务：会话推荐、组合 / 集合推荐
    

> 补充：**会话推荐**是结合多轮对话交互的推荐范式，机器通过和用户一问一答逐步摸清需求、动态调整推荐结果，核心是「边聊边推」，属于多轮交互下的序列推荐延伸。

#### 1.4.3 方法路线

- 非个性化：流行度与规则
    
- 信息来源：内容推荐、协同过滤、混合推荐
    
- 模型与表示：矩阵分解、深度、序列、图、多模态、语言模型
    
- 可组合机制：对比学习、频域、扩散、生成式检索
    

> 同一种方法可以同时具有多个方向标签。

#### 1.4.4 系统阶段与核心研究问题

- **系统三阶段**：召回（粗筛）→ 排序（精排）→ 重排（业务调整）
    
- **核心研究问题**：稀疏与冷启动、长期兴趣与短期兴趣、多样性 / 新颖性 / 可解释性、隐私 / 公平 / 鲁棒性 / 计算效率
    

## 二、推荐数据与经典方法

### 2.1 推荐系统的核心数据要素

- **用户**：人、账号、设备或匿名会话
    
- **物品**：商品、视频、音乐、论文、地点或广告
    
- **交互**：浏览、点击、收藏、购买、评分或播放
    
- **上下文**：时间、地点、设备、入口和当前会话
    

常见数据表示形式：用户 - 物品评分矩阵、用户 - 物品隐式反馈矩阵、按时间排列的行为序列。

### 2.2 用户反馈：显式 vs 隐式

|   |   |   |   |
|---|---|---|---|
|类型|定义|典型例子|特点|
|显式反馈|用户明确表达的偏好评分|星级评分、喜欢 / 不喜欢、明确标签|数据量少、标注准确|
|隐式反馈|通过用户行为间接推断偏好|曝光、点击、停留、收藏、购买、跳过|数据量大、反馈强度不一，无交互≠不喜欢|

反馈强度差异：购买通常比浏览的反馈强度更高；点击不一定代表真正喜欢，行为还会受价格、库存、场景等因素影响。

### 2.3 交互数据与任务构造

- 评分数据 → 评分预测任务
    
- 点击、收藏、购买数据 → 交互预测、Top-N 推荐任务
    
- 带时间顺序的行为数据 → 下一物品预测、会话推荐任务
    

> 同一数据集可以构造不同任务；过滤规则、时间划分和样本构造需要与研究问题保持一致。

### 2.4 负采样

#### 2.4.1 核心背景与基础概念

**采样**：从规模很大的总体集合中，按特定规则挑选一部分样本使用，在保留代表性的前提下降低计算成本。

隐式反馈数据只记录发生过的正向行为，没有完整标注的负例。用户未交互的物品，可能是用户不喜欢，也可能是用户从未看到过。负采样就是从未交互物品中挑选样本作为负例的过程。

#### 2.4.2 训练负采样

- 定义：模型训练阶段，从未交互物品中选择一部分作为负样本，与正样本共同组成训练集，让模型学习区分用户感兴趣与不感兴趣的物品。
    
- 本质：采样方式是一种**建模假设**，人为设定「这些物品可代表用户不感兴趣的内容」，直接影响模型学到的决策边界。
    
- 关键误区：不能把所有未交互物品都当作负例，其中包含大量用户未曝光的潜在兴趣物品。
    

#### 2.4.3 训练负采样 vs 评价候选集（核心辨析）

二者都可能从未交互物品中取样，但绝对不能混为一谈：

|   |   |   |
|---|---|---|
|比较维度|训练负采样|评价候选集|
|所处环节|模型训练阶段|模型验证 / 测试阶段|
|核心目的|构造正负训练样本，让模型学习决策边界|划定排序范围，在统一口径下计算推荐准确率指标|
|影响对象|模型的收敛效果、学到的偏好边界|指标数值高低、实验结果的可比性|
|通俗类比|模型的「练习题」|模型的「考试范围」|

> 补充：评价候选集是离线评测环节划定的物品排序范围，通常由「用户真实交互正样本 + 采样的未交互物品」构成，模型仅在该集合内打分排序并计算指标。候选集大小、采样方式不同，同一个指标数值不具备可比性。

### 2.5 经典推荐方法

#### 2.5.1 流行度推荐

- 基本思路：统计全体用户的点击、购买、播放量，按全局热度形成推荐列表
    
- 使用信息：仅依赖全局交互统计，不依赖个人历史行为
    
- 适用场景：新用户冷启动、缺少个性化数据、实验简单基线
    
- 主要问题：无个性化、强化热门偏置、压缩长尾物品曝光
    

#### 2.5.2 内容推荐

- 基本思路：根据用户过去喜欢的物品建立兴趣表示，推荐内容特征相似的其他物品
    
- 使用信息：物品的类别、标签、文本、图片等特征 + 用户历史喜欢的物品
    
- 优势：推荐依据直观，可利用新物品的内容信息，缓解物品冷启动
    
- 主要问题：依赖物品特征质量，容易重复推荐相似内容，造成兴趣收窄
    

#### 2.5.3 协同过滤（Collaborative Filtering, CF）

协同过滤是完全基于用户 - 物品交互行为的推荐范式，核心逻辑是「相似用户有相似偏好 / 相似物品会被相似用户喜欢」，无需依赖物品内容属性。分为两大分支：

##### UserCF（基于用户的协同过滤）

- 核心思路：找到与目标用户兴趣相似的用户群体，汇总邻居喜欢而目标用户尚未交互的物品做推荐，即「人以群分」
    
- 推荐流程：用户 - 物品交互 → 计算用户相似度 → 选择 Top-N 相似邻居 → 聚合邻居行为生成候选
    
- 优势：利用群体协同信号，逻辑直观
    
- 局限：交互稀疏时相似度失真严重；用户冷启动效果差；大规模用户下近邻计算成本高
    

##### ItemCF（基于物品的协同过滤）

- 核心思路：基于用户共同交互学习物品相似度，给用户推荐和历史喜欢物品相似的其他物品，即「喜欢 A 的人也喜欢 B」
    
- 推荐流程：用户 - 物品交互 → 计算物品共现相似度 → 匹配用户历史物品 → 聚合生成候选
    
- 优势：物品相似度相对稳定，可解释性强，早期电商场景广泛应用
    
- 局限：新物品无交互则无法计算相似度（物品冷启动）；热门物品存在共现偏置；推荐范围易过窄
    

#### 2.5.4 矩阵分解（Matrix Factorization, MF）

- 核心思路：将稀疏的用户 - 物品交互矩阵，拆解为用户隐向量矩阵和物品隐向量矩阵，把用户和物品映射到同一个低维隐空间，通过向量点积 / 余弦相似度预测偏好
    
- 使用信息：用户 - 物品评分矩阵或隐式反馈矩阵
    
- 核心产出：用户 Embedding、物品 Embedding，通过点积打分
    
- 优势：缓解数据稀疏问题，泛化能力优于基于邻居的协同过滤
    
- 局限：用户和物品冷启动问题仍存在；隐向量可解释性有限；表示完全依赖交互数据训练
    

### 2.6 经典方法对照

|   |   |   |   |
|---|---|---|---|
|方法|推荐依据|适用入口|典型问题|
|流行度|全局热度|新用户、简单基线|热门偏置|
|内容推荐|内容相似|有物品特征、新物品|兴趣收窄|
|UserCF|相似用户|群体兴趣可迁移|稀疏与用户冷启动|
|ItemCF|相似物品|物品关系较稳定|新物品冷启动|
|矩阵分解|隐向量匹配|交互规模较大|冷启动与解释有限|

### 2.7 补充：数据稀疏性问题

#### 定义

推荐系统的用户 - 物品交互矩阵中，绝大多数单元格为空，仅极少部分有交互记录，这种数据分布特性即为稀疏性。 $$稀疏度 = 1 - \frac{交互总数量}{用户数 \times 物品数$$

#### 三类稀疏

1. **用户侧稀疏**：新注册 / 低频用户，行为极少，即用户冷启动
    
2. **物品侧稀疏**：上新 / 小众商品，交互极少，即物品冷启动
    
3. **全局稀疏**：全平台交互填充率极低，是工业界普遍现状
    

#### 核心危害

- 协同过滤相似度计算失真
    
- Embedding 学习不充分，召回排序精度下降
    
- 冷启动场景无法生成有效推荐
    
- 少量偶然行为易造成噪声干扰
    
- 长尾物品难以获得曝光
    

#### 主流解决方案

引入侧信息、内容 / 知识推荐、矩阵补全、迁移学习预训练、图神经网络（GNN）、大模型语义增强、多行为数据融合等。

## 三、现代方法与真实系统

### 3.1 现代推荐方法全景

- 基础建模工具：深度学习可用于召回、排序、序列、图和多模态推荐
    
- 核心研究对象：序列推荐、图推荐、多模态推荐
    
- 通用可组合机制：对比学习、频域、扩散、生成式检索、大模型
    
- 组合形式：频域序列推荐、多模态图推荐、大模型增强的推荐模型等
    

### 3.2 核心技术方向详解

#### 3.2.1 对比学习（Contrastive Learning）

对比学习是自监督学习的核心技术，是提升特征表示质量的通用优化手段，而非独立推荐算法。

- **核心本质**：构造正样本对（同一对象的不同形态）和负样本对（不同对象），训练让正样本对向量距离拉近、负样本对距离推远，最终得到更有区分度、更鲁棒的 Embedding。
    
- **推荐场景的价值**：缓解数据稀疏、冷启动问题，补充监督信号，削弱噪声对表示的干扰，提升模型泛化性。
    
- **正负样本构造方式**：
    
    - 序列增强：对同一行为序列做掩码、截断、轻量重排，得到两条子序列互为正样本
        
    - 图结构增强：对二部图做不同邻居采样、边丢弃，同一节点的两个视图互为正样本
        
    - 多模态增强：同一物品的文本、图片等不同模态表示互为正样本
        
- **核心局限**：人为设计的数据增强可能偏离用户真实偏好，负采样策略也会显著影响效果。
    

#### 3.2.2 频域推荐

属于序列推荐的技术分支，核心是通过傅里叶变换将时域行为序列转换到频域空间，通过过滤频率成分优化推荐效果。

##### 基础概念辨析

- **时域信号**：以时间为横轴，记录行为随时间的变化过程，保留完整先后顺序。用户按时间排列的行为序列就是典型时域信号。
    
- **频域信号**：将时域信号拆解为不同频率的周期波动叠加，横轴为频率，纵轴为成分强度，不关心时间先后，只关注内在波动规律。
    
    - 高频成分：波动快、持续短，对应偶然误点、随机浏览等噪声
        
    - 中频成分：短期波动，对应用户近期即时兴趣
        
    - 低频成分：波动慢、持续久，对应用户长期稳定偏好与周期性行为
        
- 「高频 = 噪声」是核心建模假设：真实兴趣有连续性，不会瞬间突变；而噪声是转瞬即逝的突刺信号，对应高频分量。该假设并非绝对成立。
    

##### 方法详情

- 核心思路：过滤序列中的噪声与偶然点击，保留有效兴趣信号，再转回时域做预测
    
- 代表方法：
    
    - FMLP-Rec：可学习频域过滤器 + 全 MLP 结构
        
    - FEARec：联合建模时域依赖、频率成分和周期行为
        
- 常见任务：序列推荐、下一物品预测
    
- 主要问题：频率成分的推荐语义不直观，并非所有数据都存在稳定周期，效果收益依赖数据特性
    

#### 3.2.3 扩散推荐

- 核心思路：对交互或目标物品表示逐步加入噪声，学习反向去噪过程，恢复用户偏好或目标物品表示
    
- 代表方法：
    
    - DiffRec：对用户 - 物品交互向量进行去噪
        
    - DiffuRec：生成序列中的目标物品表示
        
- 常见任务：Top-N 推荐、交互去噪、下一物品预测
    
- 主要问题：多步去噪增加训练推理成本，对噪声过程、步数和实验设置较敏感
    

#### 3.2.4 生成式检索

新型召回范式，将推荐召回从「向量匹配搜索」转化为「序列生成任务」。

##### 核心概念

- **语义离散 ID**：给每个物品构造的带语义信息的离散符号编码，通常由多个 token 组成，通过向量量化得到。相似物品的语义 ID 更接近，自带语义关联。它是离散符号，不是连续向量，性质与文字 token 一致。
    
- **自回归生成**：用序列的历史值预测未来值，每一步的输出作为下一步的输入，逐个生成序列元素。生成式检索就是自回归逐个生成物品语义 ID 的 token。
    

##### 范式对比

- 传统向量检索：用户 / 查询向量 → 近似最近邻（ANN）搜索 → 候选物品
    
- 生成式检索：历史语义 ID 序列 → 自回归生成 ID → 目标物品
    

##### 详情

- 代表方法：TIGER（NeurIPS 2023）
    
- 常见任务：候选召回、下一物品预测
    
- 主要问题：语义 ID 与码本设计难度高，物品目录更新麻烦，存在解码错误风险，推理效率待优化
    

##### 与矩阵分解的核心区别

|   |   |   |
|---|---|---|
|对比维度|矩阵分解（向量检索范式）|生成式检索|
|核心范式|匹配式检索，向量化后做相似度搜索|生成式召回，转化为序列生成任务直接输出物品 ID|
|物品表示|连续低维向量（Embedding）|离散的语义 ID 序列|
|检索方式|近似最近邻（ANN）向量搜索|自回归生成解码|
|技术定位|传统协同过滤基础方法|现代新型召回范式，易与大模型打通|

#### 3.2.5 语言模型推荐

- 核心思路：将用户、物品、交互、评论和任务指令全部转换为**自然语言序列**，用统一的语言建模目标处理多种推荐任务
    
- 自然语言序列：由人类日常语言按先后顺序排列形成的文本串，自带语义信息，顺序敏感，语序改变语义可能完全不同
    
- 代表方法：P5（预训练、个性化提示和预测）
    
- 支持任务：推荐列表生成、评分预测、推荐解释、评论生成
    
- 主要问题：物品 ID 与语言语义对齐困难，生成结果约束性弱，不同任务的统一评价难度大
    
- 注意：P5 是推荐任务的语言模型化，不等同于通用对话式大模型
    

#### 3.2.6 大模型推荐

大模型在推荐系统中有三类典型应用方式：

1. **作为推荐器 / 生成器**：直接生成推荐结果、解释或对话内容
    
2. **作为对齐后的推荐模型**：用推荐数据微调通用大模型，代表方法 TALLRec
    
3. **作为知识与特征增强器**：补充物品属性、用户画像和关系信息，代表方法 LLMRec
    

- 核心价值：强大的语言理解能力、外部知识储备、跨任务表达能力、自然语言交互能力
    
- 主要问题：推理成本高、时延大、存在幻觉问题、物品落地约束多、隐私与评价难度大
    

### 3.3 现代方法对照

|   |   |   |   |
|---|---|---|---|
|方向|建模重点|常见任务|主要问题|
|对比学习|自监督训练信号|序列表征、推荐排序|正样本或增强可能偏离真实偏好|
|频域|序列频率成分|下一物品预测|周期不稳定、解释有限|
|扩散|生成与去噪|Top-N、下一物品预测|多步推理、设置敏感|
|生成式检索|物品语义 ID 生成|召回、下一物品预测|码本、更新和解码效率|
|大模型推荐|语言知识与任务接口|推荐、解释、对话、增强|成本、幻觉、落地和评价|

### 3.4 多阶段推荐系统架构

工业级推荐系统普遍采用「召回 - 排序 - 重排」三级流水线，本质是效果与性能的权衡。

#### 3.4.1 召回层

- 核心作用：从百万级全量物品池中快速粗筛出数百到数千个候选，缩小后续排序范围
    
- 设计原则：速度优先、覆盖率优先，宁可错捞不可漏掉
    
- 主流技术路线：流行度召回、协同过滤召回、向量检索召回、序列模型召回、生成式检索
    
- 工业界通常采用多路召回并行，合并去重后送入排序层
    

##### 补充：近似最近邻（ANN）搜索

向量召回的核心技术。目标是找出与用户向量匹配度最高的物品，但不做全量暴力计算，而是通过提前构建向量索引，只在少量相似分组内计算，牺牲极小精度换取几十上百倍的速度提升，是工业级海量向量检索的标配。

- 暴力精确搜索：全量物品挨个计算相似度，准但慢，仅适用于极小数据集
    
- ANN：建索引缩小搜索范围，近似准确但速度极快，适配百万级以上物品库
    
- 常见实现：IVF 倒排索引、HNSW 层次化小世界图、LSH 局部敏感哈希
    

#### 3.4.2 排序层

- 核心作用：对召回的数千候选做精细化打分，按相关性排序，输出数十到数百个结果
    
- 输入特征：综合用户、物品、交互、上下文等多维度特征
    
- 典型模型：DeepFM
    

##### 补充：DeepFM

排序阶段经典深度预估模型，将因子分解机（FM）与深度神经网络（DNN）端到端结合，同时学习低阶和高阶特征交叉。

- 结构：共享 Embedding 层 + FM 分支（一阶 + 二阶特征交叉） + DNN 分支（高阶非线性交叉） + 输出层
    
- 优势：自动特征交叉，无需人工构造，低阶高阶全覆盖，对稀疏特征友好
    
- 适用场景：CTR/CVR 预估、推荐精排
    

#### 3.4.3 重排层

- 核心作用：在排序结果基础上，加入多样性、新颖性、业务规则、安全合规、重复控制等约束，调整出最终展示列表
    
- 特点：不侧重相关性精度，侧重整体列表的业务价值与用户体验
    

#### 3.4.4 概念补充：回归与自回归

- **回归**：用已知特征预测连续数值结果，找到输入与输出的量化关系。推荐中评分预测、CTR 概率预估都属于回归任务。
    
- **自回归**：回归的特殊形式，用序列自身的历史值预测未来值，严格依赖先后顺序，每一步输出作为下一步输入。序列推荐、生成式检索、大模型文本生成都基于自回归思路。
    

### 3.5 研究模型 vs 真实工业系统

- 论文研究：单任务、单模型、单系统阶段
    
- 真实系统：多路召回 + 多模型融合，叠加规则、安全与内容治理，需满足实时性、延迟、稳定性要求
    
- 核心结论：**离线指标提高不等于用户体验一定提高**，单一模型结果不代表完整系统效果，阅读论文需明确研究对象和系统位置。
    

### 3.6 推荐系统合规

#### 合规基础要求

- 在合法依据和明确目的下使用必要数据
    
- 清晰告知用户个性化推荐规则
    
- 提供关闭个性化推荐的便捷选项
    
- 允许用户修改、删除个人标签
    

#### 高风险 / 禁止做法

- 过度收集、超范围使用、非法共享个人信息
    
- 基于用户画像实行不合理差别定价
    
- 诱导沉迷、过度消费，向未成年人推送有害内容
    
- 操纵榜单、制造虚假流量
    
- 推荐违法信息、利用算法规避监管
    

#### 法规依据

《个人信息保护法》《互联网信息服务算法推荐管理规定》《电子商务法》

## 四、推荐实验范式

### 4.1 完整实验流程

明确任务 → 选择数据 → 数据划分 → 构造样本 → 训练模型 → 选择基线 → 统一评价 → 公平比较 → 边界结论

> 模型只是实验链条的一个环节，数据、样本和比较条件同样影响结论可信度。

### 4.2 任务、数据与划分

#### 任务定义

明确是评分预测、Top-N 推荐还是下一物品预测。

#### 数据说明

需明确用户、物品、交互类型，时间信息、数据规模、稀疏程度，以及过滤和预处理规则。同名数据集经过不同处理，会形成不同的实验问题。

#### 数据划分

- **训练集**：用于学习模型参数
    
- **验证集**：用于模型选择和超参数调整
    
- **测试集**：用于最终效果评价
    

### 4.3 时间信息泄漏

#### 错误做法

随机打散全部序列行为、未来点击进入训练数据、使用未来统计量预测过去行为，都会导致指标虚高，结论失真。

#### 正确做法

严格按时间顺序划分：用过去的行为训练，较新的行为验证，最新的行为测试，与真实预测场景保持一致。

#### 其他泄漏来源

物品热度全局统计、特征构造、负样本池等。

### 4.4 实验公平比较原则

#### 基线的作用

回答「新方法究竟比什么更好」，基线需覆盖从简单到现代的多个层次：

1. 流行度推荐
    
2. ItemCF
    
3. 矩阵分解
    
4. 合适的现代推荐模型
    

#### 公平比较的一致条件

所有对比模型必须使用完全一致的计算口径，包括：

1. 相同的数据版本、预处理规则、数据划分方式
    
2. 相同的评价候选集大小、采样方式，训练负采样口径一致
    
3. 相同的推荐列表长度 K、指标计算公式
    
4. 相同的训练预算、调参条件
    

#### 不公平做法

只选择弱基线、使用不合理的基线实现、在不同实验条件下对比指标。

> 核心结论：数据划分、负样本数量、K 值任意一项不同，实验结果都不能直接横向比较。

### 4.5 核心评价指标

#### 准确性指标

|   |   |
|---|---|
|指标|含义|
|Recall@K|用户真实相关的物品中，有多少被排进了推荐列表前 K 位|
|HitRate@K|推荐列表前 K 位中，是否至少命中了一个目标物品|
|NDCG@K|同时衡量命中情况与命中位置的靠前程度，位置越靠前得分越高|
|RMSE / MAE|预测评分与真实评分的误差，用于评分预测任务|

#### 其他维度指标

覆盖率、多样性、新颖性、用户满意度、效率、鲁棒性。

> 指标必须与任务和评价候选集相对应。

### 4.6 实验结论的边界

#### 可以说明

模型在当前数据、任务、指标和实验设置下表现更好。

#### 不能自动推广

1. 一个数据集的结果 ≠ 所有领域都适用
    
2. 准确性提高 ≠ 多样性、效率同步提高
    
3. 离线指标提高 ≠ 线上用户体验一定提高
    

#### 结论要求

必须写清数据、任务、基线和实验条件，明确说明结果的适用范围与限制。

### 4.7 补充：核心概念辨析

#### 4.7.1 离线 vs 在线 / 线上

推荐领域的「离线」不是指断网，而是与「在线 / 线上」相对的概念：

- **离线**：用历史数据批量训练模型、用历史测试集计算指标，不实时服务用户，对应「离线训练」「离线评测」
    
- **在线 / 线上**：模型部署到真实产品中，实时响应用户请求，直接影响真实用户体验
    

离线是模拟考试，线上是实战检验，二者效果可能存在偏差。

#### 4.7.2 计算口径

计算指标时统一约定的计算范围、规则前提、统计标准和计算方法，即「算账的统一规矩」。

- 范围口径：评价候选集的大小
    
- 数据口径：预处理、划分规则
    
- 指标口径：公式、K 值
    
- 训练口径：负采样策略、训练条件
    

口径不同，同名指标的数值不具备可比性。
